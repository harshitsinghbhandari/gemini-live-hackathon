# Composio Python SDK Guide for Aegis

This guide covers the integration of Composio into **Aegis**, a Mac AI Agent. It focuses on pure Python implementation using API keys, specifically for Gmail and Google Calendar integrations.

## 1. Installation

Install the Composio SDK and any relevant provider packages (e.g., OpenAI, if you use it for agent logic):

```bash
pip install composio
```

## 2. Initialization

Initialize the SDK using your API key. No CLI login is required.

```python
from composio import Composio

# Initialize with API key explicitly
composio = Composio(api_key="YOUR_COMPOSIO_API_KEY")

# Alternatively, if COMPOSIO_API_KEY is an environment variable:
# composio = Composio()
```

## 3. Connecting Integrations (Gmail & Google Calendar)

To connect a new app like Gmail or Google Calendar purely via Python, use the `authorize` method. Since these use OAuth, it returns a redirect URL for the user.

```python
def connect_new_app(user_id: str, toolkit_slug: str):
    # Initiate authorization flow
    connection_request = composio.toolkits.authorize(
        user_id=user_id,
        toolkit=toolkit_slug
    )
    
    # Return this URL to the user to authorize in their browser
    print(f"Please visit: {connection_request.redirect_url}")
    
    # Optional: Block and wait for the user to complete authorization
    try:
        connected_account = connection_request.wait_for_connection(timeout=300)
        print(f"Successfully connected {toolkit_slug} for user {user_id}")
        return connected_account
    except Exception as e:
        print(f"Connection failed or timed out: {e}")
        return None

# Usage
# connect_new_app("harshitbhandari0318", "gmail")
# connect_new_app("harshitbhandari0318", "googlecalendar")
```

## 4. Checking Connection Status

Before executing a tool, you should verify if the integration is active.

```python
def is_connected(user_id: str, toolkit_slug: str) -> bool:
    # List connected accounts for this user and toolkit
    accounts = composio.connected_accounts.list(
        user_ids=[user_id],
        statuses=["ACTIVE"]
    )
    
    # Check if any active account belongs to the requested toolkit
    for account in accounts.items:
        # Note: toolkit info is nested in the auth_config or connection_data
        if account.toolkit.slug == toolkit_slug:
            return True
    return False

# Usage
# if is_connected("harshitbhandari0318", "gmail"):
#     print("Gmail is ready!")
```

## 5. Listing Available Tools

To find exact slugs and required arguments for a toolkit:

```python
def list_tools(toolkit_slug: str):

# list_tools("gmail")
```

## 6. Executing Tools & Syntax

Tool execution follows a consistent syntax. You pass the `slug`, `arguments` (dictionary), and the `user_id`.

```python
response = composio.tools.execute(
    slug="GMAIL_FETCH_EMAILS",
    arguments={"q": "from:support@github.com", "max_results": 5},
    user_id="harshitbhandari0318"
)

# Response structure
if response["successful"]:
    data = response["data"]
    print("Execution Result:", data)
else:
    print("Error:", response["error"])
```

### Response Object Fields
- `successful` (bool): `True` if the tool executed without server-side error.
- `data` (dict): The actual content returned by the integration (e.g., list of emails).
- `error` (str | None): Error message if `successful` is `False`.

## 8. Error Handling

Handle common SDK and connection errors gracefully.

```python
from composio import exceptions

try:
    response = composio.tools.execute(...)
except exceptions.ToolNotFoundError:
    print("The requested tool slug does not exist.")
except exceptions.ComposioSDKTimeoutError:
    print("The operation timed out.")
except exceptions.ComposioError as e:
    print(f"A Composio error occurred: {e}")
```

## 9. Gotchas and Known Issues

- **Toolkit Versions**: Usually, the SDK uses the "latest" version. If execution fails asking for a version, pass `version="latest"` or a specific timestamp version found in `tool.available_versions`.
- **Primary Calendar**: For Google Calendar, `calendar_id` usually defaults to `"primary"`, but it's safer to provide it explicitly.
- **User ID Persistence**: Composio uses the `user_id` to map to `connected_account_id`. Ensure `"harshitbhandari0318"` is used consistently to avoid re-authentication prompts.
- **Rate Limits**: The SDK handles retries automatically (default 2 retries). You can adjust this in `SDKConfig` via `max_retries`.
- **Response Format**: Results derived from `data` can be deeply nested depending on the underlying API (Gmail/Google Calendar). Always verify the structure by printing `response['data']` once.
